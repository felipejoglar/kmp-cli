# Releasing

This document describes how to cut a new release of KMP CLI.

## Prerequisites

- On the `main` branch with a clean working tree, synced with `origin/main`
- Java 17+ installed

## Pre-release Checklist

1. Ensure CI is green on `main`
2. Run local checks:
   ```bash
   ./gradlew compileKotlinLinuxX64 allTests
   ./bin/verify
   ```
3. Choose a version number following [semver](https://semver.org/) (e.g. `1.2.3`, no `v` prefix)

## Cutting a Release

Run the release script with the desired version (no `v` prefix):

```bash
./bin/release 1.2.3
```

The script validates that:
- The version is valid semver
- The tag doesn't already exist
- You're on the `main` branch
- The working tree is clean
- Local `main` is in sync with `origin/main`

After confirmation, it creates an annotated tag and pushes it. Use `DRY_RUN=1` to verify without pushing:

```bash
DRY_RUN=1 ./bin/release 1.2.3
```

The tag push triggers the [release workflow](.github/workflows/release.yml), which runs these jobs in order:

1. **Test** — compiles and runs `allTests` on Linux
2. **Build** — produces native binaries for all targets (macosArm64, linuxX64, linuxArm64) with the version stamped via `-PcliVersion`
3. **Release** — creates a GitHub Release with archives, checksums, and auto-generated release notes
4. **Smoke Test** — downloads the published archives on macOS and Linux, verifies checksums, and runs `--version`

## Version Stamping

- The version is extracted from the git tag: `${GITHUB_REF_NAME#v}` → passed as `-PcliVersion=X.Y.Z`
- A `BuildConfig.kt` file is generated at build time with the embedded version constant
- Local dev builds default to `0.0.1`

## Artifacts

Each release publishes the following to GitHub Releases:

| Artifact | Contents |
|---|---|
| `kmp-cli-darwin-arm64.tar.gz` | macOS Apple Silicon binary + templates |
| `kmp-cli-linux-amd64.tar.gz` | Linux x86_64 binary + templates |
| `kmp-cli-linux-arm64.tar.gz` | Linux ARM64 binary + templates |
| `checksums.txt` | SHA-256 checksums for all archives |

## Installation

Users install the latest release via:

```bash
curl -fsSL https://raw.githubusercontent.com/felipejoglar/kmp-cli/main/scripts/installer.sh | bash
```

The installer detects OS/architecture, downloads the correct archive, verifies the checksum, and installs to `~/bin`.

## Changelog

Release notes are auto-generated by GitHub using the categories defined in [`.github/release.yml`](.github/release.yml):

- ⚠️ Breaking Changes (`breaking` label)
- Features (`enhancement` label)
- Bug Fixes (`bug` label)
- Documentation (`documentation` label)
- Other Changes

Dependabot PRs are excluded automatically.

## Rollback

If a release needs to be pulled:

1. Delete the GitHub Release from the [releases page](https://github.com/felipejoglar/kmp-cli/releases)
2. Delete the tag:
   ```bash
   git tag -d v<VERSION>
   git push origin --delete v<VERSION>
   ```
3. Fix the issue on `main`
4. Re-tag and push with the same or a new version number
